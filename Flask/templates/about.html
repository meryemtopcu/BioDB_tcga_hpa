About My BioDB

My BioDB is a lightweight educational web application that integrates TCGA-BRCA RNA-seq gene expression with Human Protein Atlas (HPA) protein expression summaries, and exposes them through a searchable MySQL-backed website.

Why it was built (learning goals)

  • Database design & loading: build relational tables and import large biological datasets.
  • SQL querying: filter, aggregate, and summarize gene- and sample-level information.
  • Backend: connect user input to database queries using Flask routes and helper functions.
  • Frontend: present results with Bootstrap tables and Chart.js visual summaries.

Data sources in this prototype

  • TCGA-BRCA (RNA expression): RNA-seq quantification per sample (e.g., counts/TPM/FPKM depending on the imported columns).
  • Human Protein Atlas (HPA): gene-level protein detection categories (High / Medium / Low / Not detected) for a cancer type.

Note: this project uses a subset of rows/columns imported into MySQL for learning and demonstration.

What this app shows

  • TCGA: per-sample RNA expression values for the selected gene ID.
  • HPA: gene-level protein detection summary (if available for that gene).
  • Charts: quick visual overview of HPA categories and TCGA TPM per sample.
  • External links: pointers to NCBI Gene, Ensembl, and HPA for further validation.

Gene symbol vs. Gene ID (why “Resolve” exists)

Users search by gene symbol (e.g., MATR3), but the database queries are performed using stable Ensembl Gene IDs (ENSG...).

  • Some symbols map to multiple Ensembl Gene IDs (annotation differences) → the Resolve page lets you choose.
  • A gene can have TCGA RNA data but no HPA entry (protein table coverage is a subset).
  • Some resolved IDs can show 0 expression in this dataset; choosing another candidate ID may change the result.

Example gene with both sources available: MATR3

Technical stack

  • Backend: Flask (Python)
  • Database: MySQL
  • Frontend: Bootstrap 4
  • Charts: Chart.js
  • Config: YAML for database credentials

Possible future extensions

  • Add additional cancer types / cohorts and a cancer-type filter.
  • Add a curated gene symbol ↔ Ensembl ID mapping table to reduce ambiguity.
  • Add normalization/summary metrics (e.g., percentiles, z-scores) and better comparisons across samples.
  • Add pagination and performance optimizations for large result sets.
  • Expose a small JSON API endpoint for programmatic use.
